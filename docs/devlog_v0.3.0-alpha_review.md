# 目标

统一唐宋词语料。

## 诗词语句的相似检索
输入一句词句，系统返回语义接近或可能被化用的诗句
- 单句（分割标点小于句号）：可能相近语句太多，模糊重点
- 多句（分割标点大于等于句号）：需要整句都构建向量库，工程量似乎很大
	- 对于词句，可能考虑不按照顿号分割
	- 如果甚至合并逗号短句（比如连续三个三字句）可能命中率更高
## 交互式词集阅读器
提供若干作品的在线阅读，点击某一句可展开「可能的来源诗句」
- 选择最近的文字（确保这一句作者生卒年早于当前作品的作者），比较直接，但可能缺乏公共源头
- 嵌套搜索，挖到源头（可能最早出处都是一些老生常谈的作品如诗经楚辞），可能文字扭曲、信息损失
	- 如何处理化用链
		- 按生卒年加权
		- 提取意象对，如果存在关键意象对则加分
		- 引用计数

## 典故、意象、注释
- 典故可能需要引入非诗词作品（古文、志怪小说）
- 意象可能需要对语料库进行统计收集清洗
- 注释可能考虑离线词典（古汉语词典、辞海、词源）或在线爬取（缺乏爬取能力）

## 文学评论
- 词评：需要录入大量词评，设计rag方案
- 作者生平、创作背景：难以收集全，不同作者的信息不同，来源可能有矛盾

## 赏析
结合以上所有或部分信息，由 AI 进行赏析
- 可能设计 agent 工作流编排

## 其他
知识库（典故、意象、词评）构建或模型训练
选定基模，进行大量作品阅读，沉淀出知识，便于查询
考虑使用上海古籍出版社的个人词集、唐圭璋的词选、俞陛云的词选等等（以上均有词评、注释）

## 今天发现的问题
Poetry 里宋_x.csv 不止有诗，还有词，并且词牌还有「调笑令 其一 效韦应物体」的格式（也就是说词牌+标题）
如何权衡，是否应当放弃 Poetry 里的宋代诗，毕竟对我们唐诗引用关系不大，或者说暂时放弃？
Poetry 里唐.csv 里也有唐五代词，是否应当暂时隐去 genre 字段
唐.csv 里有重复语料，比如温庭筠《更漏子》与《更漏子 其一》，前者是包含后者的套词，但两者都存在
宋.csv 有错误标记的语料，比如将含有「画屏金鹧鸪」的《更漏子》标记为苏轼作品
