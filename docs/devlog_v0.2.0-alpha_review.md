# 目标

增加对全宋词语料的语义检索支持。


## 语料处理

接入 chinese-poetry 中的全宋词数据。
数据中包含部分繁体字，使用 OpenCC 进行繁转简统一处理。

## 句子切分策略

chinese-poetry 中的全宋词数据按句号切分，而不是更细粒度。

embedding 时需要考虑三种可能策略：

- 句号级切分
- 更细粒度（按所有标点切分）
- 折中粒度

### 当前版本选择：

按所有标点切分为最细粒度句子后进行 embedding，先观察检索效果。

### 未来优化方向

如果未来需要混合多种切分粒度，可以考虑：

原子化索引 + 动态上下文展示策略

具体思路：

检索阶段：以单句作为索引单元，保证召回精度
展示阶段：通过 parent_id 与 line_idx 聚合上下文
扩展上下文窗口并去重，避免语义相似句造成冗余展示
